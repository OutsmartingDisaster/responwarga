## 1. Executive Summary

Sistem manajemen operasi respon bencana yang memungkinkan organisasi (org_admin) untuk mengaktivasi respon pada bencana tertentu, mengelola tim responder, dan mengkoordinasikan penanganan laporan warga.

---

## 2. Problem Statement

Saat ini sistem hanya mendukung:
- Laporan warga (butuh bantuan / beri bantuan)
- Organisasi dan anggota

Yang belum ada:
- Mekanisme aktivasi respon bencana oleh organisasi
- Koordinasi tim responder di lapangan
- Laporan lapangan dari tim responder
- Assignment laporan warga ke responder
- **Auto-dispatch laporan warga ke org_admin berdasarkan lokasi**
- **Super admin untuk oversight semua data**

---

### 4.5 Auto-Dispatch System

Ketika warga submit laporan (minta bantuan / beri bantuan) dari public page:

**Dispatch Logic:**

```
1. Ambil koordinat laporan warga
2. Cari response_operations yang:
   - status = 'active'
   - Lokasi laporan dalam radius disaster_radius_km dari disaster_lat/lng
3. Jika ditemukan:
   - Dispatch ke org_admin dari organization yang punya operasi tersebut
   - Kirim notifikasi ke org_admin
   - Set report.dispatched_to = organization_id
   - Set report.dispatched_at = NOW()
4. Jika tidak ditemukan operasi aktif:
   - Laporan tetap tersimpan (visible di super admin)
   - Status: 'unassigned'
```

**Dispatch Status pada Reports:**

- `unassigned` - Belum ada operasi aktif di area tersebut
- `dispatched` - Sudah di-dispatch ke organisasi
- `acknowledged` - Org admin sudah melihat
- `assigned` - Sudah di-assign ke responder
- `in_progress` - Sedang ditangani
- `resolved` - Selesai ditangani

**Multiple Operations Scenario:**

Jika ada lebih dari satu operasi aktif yang cover lokasi yang sama:
- Dispatch ke operasi yang paling dekat (jarak ke center point)
- Atau dispatch ke semua (configurable by super admin)

## 5. Database Schema

### 5.1 New Tables

```sql
-- Response Operations
CREATE TABLE response_operations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID REFERENCES organizations(id) ON DELETE CASCADE,
  name VARCHAR(255) NOT NULL,
  disaster_type VARCHAR(50) NOT NULL,
  description TEXT,
  disaster_location_name VARCHAR(255) NOT NULL,
  disaster_lat DECIMAL(10, 8) NOT NULL,
  disaster_lng DECIMAL(11, 8) NOT NULL,
  disaster_radius_km DECIMAL(5, 2) DEFAULT 10,
  posko_name VARCHAR(255),
  posko_address TEXT,
  posko_lat DECIMAL(10, 8),
  posko_lng DECIMAL(11, 8),
  status VARCHAR(50) DEFAULT 'active',
  created_by UUID REFERENCES users(id),
  started_at TIMESTAMP DEFAULT NOW(),
  ended_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Response Team Members
CREATE TABLE response_team_members (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  response_operation_id UUID REFERENCES response_operations(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  role VARCHAR(50) DEFAULT 'responder',
  status VARCHAR(50) DEFAULT 'invited',
  invited_by UUID REFERENCES users(id),
  invited_at TIMESTAMP DEFAULT NOW(),
  joined_at TIMESTAMP,
  UNIQUE(response_operation_id, user_id)
);

-- Field Reports
CREATE TABLE field_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  response_operation_id UUID REFERENCES response_operations(id) ON DELETE CASCADE,
  reported_by UUID REFERENCES users(id),
  category VARCHAR(50) NOT NULL,
  subcategory VARCHAR(100),
  title VARCHAR(255) NOT NULL,
  description TEXT,
  location_name VARCHAR(255),
  latitude DECIMAL(10, 8),
  longitude DECIMAL(11, 8),
  severity VARCHAR(50),
  urgency VARCHAR(50),
  affected_count INTEGER,
  quantity_delivered VARCHAR(255),
  photos TEXT[] DEFAULT '{}',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Report Assignments
CREATE TABLE report_assignments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  report_id UUID REFERENCES reports(id) ON DELETE CASCADE,
  response_operation_id UUID REFERENCES response_operations(id) ON DELETE CASCADE,
  assigned_to UUID REFERENCES users(id),
  assigned_by UUID REFERENCES users(id),
  status VARCHAR(50) DEFAULT 'pending',
  priority VARCHAR(50) DEFAULT 'normal',
  notes TEXT,
  response_notes TEXT,
  assigned_at TIMESTAMP DEFAULT NOW(),
  accepted_at TIMESTAMP,
  started_at TIMESTAMP,
  completed_at TIMESTAMP,
  UNIQUE(report_id, assigned_to)
);

-- Notifications
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  type VARCHAR(100) NOT NULL,
  title VARCHAR(255) NOT NULL,
  message TEXT,
  reference_type VARCHAR(50),
  reference_id UUID,
  read_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 7. API Endpoints

### 7.1 Response Operations

```
GET    /api/operations              - List operations (filtered by org)
POST   /api/operations              - Create new operation
GET    /api/operations/[id]         - Get operation detail
PATCH  /api/operations/[id]         - Update operation
PATCH  /api/operations/[id]/status  - Update operation status
```

### 7.2 Team Members

```
GET    /api/operations/[id]/members       - List team members
POST   /api/operations/[id]/members       - Invite member
PATCH  /api/operations/[id]/members/[uid] - Update member status (accept/decline)
DELETE /api/operations/[id]/members/[uid] - Remove member
```

### 7.3 Field Reports

```
GET    /api/operations/[id]/field-reports     - List field reports
POST   /api/operations/[id]/field-reports     - Create field report
GET    /api/field-reports/[id]                - Get field report detail
```

### 7.4 Assignments

```
GET    /api/assignments                  - List my assignments (responder)
GET    /api/operations/[id]/assignments  - List assignments in operation (admin)
POST   /api/operations/[id]/assignments  - Create assignment
PATCH  /api/assignments/[id]             - Update assignment status
```

### 7.5 Super Admin

```
GET    /api/admin/reports              - All reports (with filters)
GET    /api/admin/operations           - All operations
GET    /api/admin/organizations        - All organizations
PATCH  /api/admin/organizations/[id]   - Update org status
GET    /api/admin/analytics            - Get analytics data
GET    /api/admin/analytics/reports    - Reports statistics
```

### 7.6 Dispatch

```
POST   /api/dispatch/process           - Process dispatch for a report (internal)
GET    /api/dispatch/config            - Get dispatch configuration

```

### 7.7 Notifications

```
GET    /api/notifications           - List my notifications
```
